As I mentioned in the beginning, SQL is used for more than just queries.

So far we've only queried information from the database. We're far from done with the SELECT command, but let's look briefly at how we can add data into the database.

Data is inserted into a table using the INSERT INTO command.

GalaXQL has a simple table called 'hilight' which can be used to hilight stars. In addition to being brighter in the star map, the hilighted star with the lowest starid is also magnified, so you can see any planets and moons that the system has.

The hilight table only has one column - starid. The column also has 'unique' restriction on it, and thus if you try to insert a value into the table that it already has, you'll get an error.

The format for the INSERT INTO command is as follows:

INSERT INTO tablename (column names) VALUES (values for the columns)


If all columns are filled, the (column names) can be dropped. So, what happens if you're not filling all the columns? The remaining ones are set to NULL. We won't have to worry about that yet, though. Try this one:

INSERT INTO hilight VALUES (12340)
(Try it!)


After that command you should see one brigher spot on the galaxy map about halfway between the center and the edge. You should also see the zoomed system display, and if you're lucky, the system will also have some planets and moons in it.

You can remove all the hilights with the command:

DELETE FROM hilight
(Try it!)


Note that this is a very dangerous command; only use it on the hilight table for now. We'll look more into DELETE FROM later on.
