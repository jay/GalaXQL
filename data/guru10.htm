The galaxy consists of three tables - stars, planets and moons. Let's look at planets.

SELECT * FROM planets
(Try it!)


Planets have planetid, starid, orbitdistance, name, color and radius.

As you may guess, the starid points to a row in stars. Not all stars have planets, but a star may have several planets.

To list all stars which have planets, you could try:

SELECT stars.starid FROM planets, stars WHERE planets.starid=stars.starid
(Try it!)


As you can see, we can SELECT from several tables at once, and make choices on what to select based on several table's fields. Like with column names, we can use aliases for table names, like this:

SELECT s.starid FROM planets AS p, stars AS s WHERE p.starid=s.starid
(Try it!)


..which saves some typing in complicated queries.

Now, when you run the above query, you will receive lots of duplicates. This is because one row is returned for each planet, and a star may have several planets. To remove duplicates, you can use the DISTINCT keyword:

SELECT DISTINCT s.starid FROM planets AS p, stars AS s WHERE p.starid=s.starid
(Try it!)


The same applies for the relationship between planets and moons. To list all the planets with moons, try:

SELECT DISTINCT p.planetid FROM planets AS p, moons AS m WHERE p.planetid=m.planetid
(Try it!)


Finally, to list all the stars with planets with moons..

SELECT DISTINCT p.starid FROM planets AS p, moons AS m WHERE p.planetid=m.planetid
(Try it!)


Nothing stops you from doing weird linking such as planets.starid=moons.planetid, but the usefulness of the results from such queries are questionable.
