SQL stands for the standard Structured Query Language.

What it actually is could be called Somewhat Standard Entry Format For More Than Just Queries, but SSEFFMTJQ is somewhat awkward, so SQL it is.

With queries the user (or a program) can ask the database for information. The simplest query you can enter looks like this:

SELECT 3
(Try it!)

If you go and try that out, you'll find that the database will return '3', which is what you asked for. (You can hop between the query results tab and this one whenever you want. I won't be going anywhere).

The database can also do some basic (and some less-basic) mathematical operations, such as addition (+), substraction (-), multiplication (*) and division (/). You can also use parenthesis to write more complicated calculations (such as (2+2)*3).

Note that the operations supported by the database vary from one database to another. So much for 'standard'..

The results will depend on the data type entered; if you write 5/2, the result will be 2. If you write 5.0/2.0 instead, the result will be 2.5.

As your very first task, use SQL to calculate the number of seconds in one day (where day is 24 hours, hour is 60 minutes and a minute is 60 seconds).
