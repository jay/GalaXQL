You can use SQL to copy information from one table to another, either directly or by modifying it on the way. The command syntax is:

INSERT INTO tablename (column names) SELECT ..

Where the SELECT bit is a full-blown SELECT query which should return the proper columns. If all columns of the target table are filled, the (column names) can be dropped. (Like before, if we're not filling all the values in the target table, the columns are most likely set to NULL).

Now we're starting to have pretty powerful commands in our hands. To hilight half the galaxy, all you need to do is:

INSERT INTO hilight SELECT starid FROM stars WHERE x > 0
(Try it!)

If you try that and get an error stating that starid is not unique, the reason is that you probably already have some hilighted stars. Running the dangerous DELETE FROM hilight before the above will fix that problem.
