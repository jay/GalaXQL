Sometimes - and hopefully not very often - circumstances arise where you need to change tables that are already in use.

Most of the time its better to create new tables and move the data from the old tables to the new ones, but it may also be possible to use the ALTER TABLE command (I don't think I need to emphasize how dangerous this command can be).

The syntax for the ALTER TABLE command is:

ALTER TABLE tablename alteration


where some of the typical alternations are: (again, supported features vary from one database engine to the next)


RENAME TO
In order to rename the whole table, you can use the RENAME TO alteration. For example:

ALTER TABLE stars RENAME TO bright_objects
(Try it!)

(This will break the database, so be sure to either run it inside a transaction that you can rollback, or rename bright_objects back to stars with a similar command).


ADD COLUMN
To add new fields to a table, you can use the ADD COLUMN alteration. For example:

ALTER TABLE hilight ADD COLUMN notes TEXT
(Try it!)


Note that this will add NULL values to every row for that column. There are some limitations on what you can add; see the reference for details.


RENAME COLUMN.. TO..
This command can be used to rename a column. SQLite does not support RENAME COLUMN. SQLite does not support MODIFY COLUMN. Example:

ALTER TABLE hilight RENAME COLUMN notes TO remarks


MODIFY COLUMN
To change the data type of a column, you could use this command. SQLite does not support MODIFY COLUMN.

ALTER TABLE hilight MODIFY COLUMN remarks CHAR(200) NOT NULL


DROP COLUMN
Removes a column from a table. SQLite does not support DROP COLUMN.

ALTER TABLE hilight DROP COLUMN remarks

Add one whole new row and change the 'moredata' value of at least one existing row.
