The database consists of tables, which consist from columns and rows.

Consider a spreadsheet..

NameLoanedInterest
Jack$505% monthly
Tom$25000Arm and leg


Here the 'Name', 'Loaned' and 'Interest' are columns, and Jack and Tom have their rows. If we call that table 'loans', we could ask for all of the above with:

SELECT * FROM loans
(Try it!)

If you tried the above, you probably received an error that there's no 'loans' table. Since we're playing with a galaxy here, and not some boring financial database, we play with stars, planets and moons. Try the following instead:

SELECT * FROM stars
(Try it!)

This kind of command typically results in lots and lots of data. Looking at the query, note the * - it's SQL-ish for "all columns". If we wanted only some of the columns, or wanted the columns in some specific order, we would list only the wanted columns here. For example:

SELECT name, class FROM stars
(Try it!)

You can also give new aliases for the columns:

SELECT name, class AS starcolor FROM stars
(Try it!)

This comes very handy when you, for instance, use some of the arithmetic operations we played with earlier:

SELECT name, ((class+7)*intensity)*1000000 AS temperature FROM stars
(Try it!)

(Note: These values have nothing to do with reality; all the data in our galaxy is purely fictional).
